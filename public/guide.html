ⓘ Overview

Pull Plan Note Creator v1.0 is a web-based construction planning tool built by AWC Technology, LLC. It lets field supervisors and project managers create pull plan sticky notes, organize tasks by trade, and export a formatted workplan directly into Autodesk Build / Forma via XLSX import or API push.

Key Capabilities

  • Create tasks organized by trade with color-coded sticky notes
  • Populate all 18 Forma/Autodesk Build workplan import fields
  • Preview the full export table before downloading
  • Auto-save your board as a named session (stored in the cloud database)
  • Export a properly formatted .xlsx file ready for Forma import
  • Push the file directly to Autodesk Build via API (when credentials are configured)
  • Manage users, trades, and session history from the Admin panel
🔒 All data is stored securely in a cloud PostgreSQL database. Sessions auto-save whenever you add or remove tasks.

🔒 Signing In

The app requires a username and password. Contact your system administrator to receive credentials.

Login Steps

Navigate to the app URL. You will be redirected to the Sign In page automatically.
Enter your Username and Password, then click Sign In.
You are taken directly to the Pull Plan Creator. Your name and email appear in the top-right corner.

Signing Out

Click Sign Out in the top bar at any time. Your board is auto-saved first, so no work is lost.

👤 Default administrator credentials are set up by your system admin. If you cannot log in, ask your admin to verify your account is active.

📝 Filling in the Form

The task form maps directly to the 18 columns in the Forma/Autodesk Build workplan import template. Required fields are marked with a red asterisk *. Optional fields are in the More Fields expandable section.

Core Fields (always visible)

Task TitleShort name for the task, e.g. "Pour Slab – Level 2"Yes
Start DatePlanned start dateYes
Duration (Days)Number of working days for the task (minimum 1)Yes
Crew SizeNumber of workers assigned (minimum 1)Yes
PriorityLow, Normal (default), or HighNo
StatusOpen (default), In Progress, Backlog, Blocked, Work Done, Complete, IncompleteNo

More Fields (click ▶ More Fields to expand)

FieldDescription

Work TypeWork, Delivery, Inspection, Roadblock, Safety, or Downtime
Complete %Percentage complete, 0–100
AssigneeProject member email address

💡 Use Handoff IDs to define task dependencies. Set a unique number in each task's "Handoff ID", then list predecessors in "Previous Handoff IDs". Autodesk Build uses these to draw dependency arrows on the schedule.

When all required fields are filled, click Add to Board. The form clears automatically so you can enter the next task.

📌 The Task Board

After clicking Add to Board, a color-coded sticky note appears below the form layout. Each card shows the trade, task title, start date, duration, and crew size at a glance. The board auto-saves to the database within 1–2 seconds of every change.

Sticky Note Cards

  • The card's top border color matches the selected trade's accent color.
  • The card's background matches the trade's background color.
  • Status and Priority are shown on the card only when they differ from the defaults (Open / Normal).
  • Click the ✕ button in the top-right of any card to remove that task.

Board Summary (right sidebar)

StatMeaning
Total TasksNumber of tasks currently on the board
Total DaysSum of all task durations
Total Crew-DaysSum of (Duration × Crew Size) across all tasks — useful for labor planning

⚡ Actions: Preview, Export & Push

The Actions card in the right sidebar contains four buttons that are enabled once tasks are on the board.

ButtonWhat it does
👁 Preview XLSX Opens a scrollable 18-column table showing exactly what will be exported — review before committing to a download or push.
↓ Export XLSX Downloads pull-plan-import.xlsx — a formatted Excel file with frozen header, auto-filter, and alternating row shading, ready to import into Autodesk Build.
▴ Push to Autodesk Build Uploads the XLSX directly to your configured Autodesk Build project folder via the APS API. No manual download needed. Requires credentials in Admin → Settings.
🗑 Clear Board Removes all tasks from the board after a confirmation prompt. The current auto-save session is unlinked so the next add starts a fresh session.

Previewing Before Export

Add your tasks to the board.
Click Preview XLSX in the Actions card. A modal opens showing all 18 columns in a scrollable table.
Review the data. If corrections are needed, close the modal, remove the task with ✕, and re-add it with the corrected values.
Click Export XLSX from the modal footer (or from the Actions card) to download.

Importing into Autodesk Build

Export the XLSX file to your computer.
Open Autodesk Build → Schedule → Work Plans.
Click Import, select the downloaded file, and confirm.
⚠ Do not rename, reorder, or delete any columns in the exported file before importing — Autodesk Build requires the exact column layout.

💾 Saving & Loading Sessions

A session is a named snapshot of the tasks currently on your board. Sessions are stored in the cloud database and persist across browser closes and device changes. When you sign back in, your most recent session is automatically restored.

Auto-Save

Whenever you add or remove a task, the board is automatically saved within 2 seconds. The first auto-save creates a session named "Auto-save [date/time]". Subsequent changes update that same session silently.

Manually Saving a Named Session

Click Save Session in the top bar.
Enter a descriptive name, e.g. "Level 3 Pour – Week 14".
Click Save. The session appears immediately in the Saved Sessions panel.

Loading a Session

Find the session in the Saved Sessions panel on the right sidebar.
Click Load. If you have tasks on the board, you will be asked to confirm before overwriting.
The board reloads with all tasks from that session. Further changes auto-save back to the same session.
⚠ Clicking Del next to a session permanently deletes it from the database. This cannot be undone. System admins can also view a full Session History log of every past save.

👤 Admin: User Management

Click the Admin button in the top bar (visible to Admin and System Admin users) to open the Admin panel. The Users tab manages who can access the app.

User Roles

RolePermissions
UserCreate tasks, save/load their own sessions, preview and export XLSX, push to Autodesk Build
AdminAll User permissions + manage users and trades via the Admin panel
Sys AdminAll Admin permissions + view all users' sessions, view Session History, configure Autodesk credentials

Adding a User

Click + Add User in the Users tab.
Enter a username (required), full name, email, password (required), and role.
Click Save User. The new user can sign in immediately.

Editing a User

Click Edit on any row. The username cannot be changed after creation. Leave the password field blank to keep the existing password.

Deactivating a User

Edit the user and set Active to No. Deactivated users cannot sign in, but their session data is preserved in the database.

Editing a Session

Click Edit (or the task count badge) to open the Edit Session modal. You can rename the session, edit any field on any task, remove individual tasks, and save the changes back to the database.

⚠ Deleting a session from Data Access permanently removes it for the user. Edits made here are saved directly to the user's live session.

🕐 Admin: Session History

Visible to System Admin users only. The Session History tab is an immutable audit log — every save event (auto-save or manual) creates a new record that is never overwritten.

What is Recorded

ColumnDescription
Saved AtExact date and time of the save event
UserUsername of the person who triggered the save
Session NameName of the session at the time of saving
Actioncreated for new sessions, updated for changes to existing sessions
TasksNumber of tasks in the snapshot at that moment
Session IDLinks the history entry to the live session in Data Access

Viewing a Snapshot

Click View on any row to open a read-only modal showing the full task list as it existed at that exact save — including title, trade, status, dates, duration, crew, priority, assignee, and description.

Clearing the History

Click Clear History to delete all history records after a confirmation prompt. The live sessions in Data Access are not affected. The history can hold up to 500 of the most recent entries.

📄 Session History is useful for auditing who saved what and when, and for recovering task data from an earlier point in time.

🛠 Admin: Settings & Autodesk Integration

Visible to System Admin users only. Configure the Autodesk Platform Services (APS) credentials to enable the Push to Autodesk Build button for all users.

One-Time Setup

Go to aps.autodesk.com, sign in, and create a new app. Enable the Data Management API.
Copy the Client ID and Client Secret from the APS app dashboard.
In this app, go to Admin → Settings and paste your credentials, Hub/Account ID, Project ID, and target Folder URN.
Click Save Settings. The status indicator turns green when credentials are present.

Settings Reference

SettingWhere to find it
Client IDAPS app dashboard at aps.autodesk.com
Client SecretAPS app dashboard (shown at creation — save it securely)
Hub / Account IDBIM 360 Account Admin page — starts with b.
Project IDAutodesk Build → Project Settings — starts with b.
Folder URNUse the APS Explorer or Data Management API to find your target upload folder URN
⚠ The Client Secret is masked after saving. Keep a secure copy — you must re-enter it if you ever need to update your credentials.
