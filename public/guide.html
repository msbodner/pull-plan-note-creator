ⓘ Overview

Pull Plan Note Creator v1.0 is a web-based construction planning tool built by AWC Technology, LLC. It lets field supervisors and project managers create pull plan sticky notes, organize tasks by trade, and export a formatted workplan directly into Autodesk Build / Forma via XLSX import or API push.

Key Capabilities

  • Create tasks organized by trade with color-coded sticky notes
  • Populate all 18 Forma/Autodesk Build workplan import fields
  • Auto-save your board as a named session (stored in the cloud database)
  • Export a properly formatted .xlsx file ready for Forma import
  • Push the file directly to Autodesk Build via API (when credentials are configured)
  • Manage users and trade color definitions from the Admin panel
🔒 All data is stored securely in a cloud PostgreSQL database. Sessions auto-save whenever you add or remove tasks.

▶ Getting Started

The app opens directly to the Pull Plan Creator. No login is required for standard use — you are automatically signed in as Admin.

Screen Layout

  • Top Bar — User Guide, Save Session, user name, Admin button, Sign Out
  • Left Panel — Trade selector and task entry form
  • Right Panel — Saved sessions list and board summary stats
  • Bottom — The task board (sticky notes), Export and Push buttons
💡 Start by selecting a Trade from the dropdown — the form will change color to match. Then fill in the task details and click Add to Board.

📋 The Task Board

After filling in the form, click Add to Board. A color-coded sticky note appears on the board below, showing the task title, start date, duration, and crew size. The board also auto-saves to the database within 1–2 seconds.

Board Actions

  • ✕ (remove) — Click the X on any sticky note to remove that task
  • Clear Board — Removes all tasks (asks for confirmation)
  • Export XLSX — Downloads a formatted Excel file (active when board has tasks)
  • Push to Autodesk Build — Uploads directly to your Autodesk project (requires credentials in Admin → Settings)

Board Summary (right panel)

  • Total Tasks — Count of tasks on the board
  • Total Days — Sum of all task durations
  • Total Crew-Days — Sum of (Duration × Crew Size) across all tasks

💾 Saving & Loading Sessions

A session is a named snapshot of the tasks currently on your board. Sessions are stored in the cloud database and persist across browser closes and device changes.

Auto-Save

Whenever you add or remove a task, the board is automatically saved within 2 seconds. The first auto-save creates a new session named "Auto-save [date/time]". Subsequent changes update the same session silently.

Manually Saving a Session

Click Save Session in the top bar.
Enter a descriptive name, e.g. "Level 3 Pour – Week 14".
Click Save. The session appears in the right panel list.

Loading a Session

Find the session in the Saved Sessions panel on the right.
Click Load. If you have unsaved tasks on the board, you will be asked to confirm before overwriting.
The board reloads with all tasks from that session. Further changes auto-save back to the same session.
⚠ Clicking Del next to a session permanently deletes it from the database. This cannot be undone.

↓ Exporting to XLSX

The Export XLSX button generates a Microsoft Excel file formatted exactly to the Forma / Autodesk Build Work Plan Import template — all 18 columns, frozen header row, auto-filter, and alternating row shading.

How to Export and Import into Forma

Add all tasks to the board.
Click Export XLSX. The file pull-plan-import.xlsx downloads to your computer.
Open Autodesk Build → Schedule → Work Plans.
Click Import and select the downloaded file.
Review the import preview and confirm.
📄 Do not rename or delete any columns in the exported file before importing — Autodesk Build requires the exact column order.

▴ Pushing to Autodesk Build

The Push to Autodesk Build button uploads the XLSX file directly to your Autodesk Build project using the Autodesk Platform Services (APS) API — no manual download required.

One-Time Setup (Admin only)

Go to aps.autodesk.com, sign in, and create a new app. Enable the Data Management API.
Copy the Client ID and Client Secret.
In this app, click Admin → Settings and paste your credentials, Project ID, and target Folder URN.
Click Save Settings. The status indicator will turn green.

Pushing a Work Plan

Ensure tasks are on the board.
Click Push to Autodesk Build and confirm the prompt.
The file is uploaded to your configured Autodesk Build project folder.
In Autodesk Build, go to Schedule → Work Plans and import from the Files folder.

👤 Admin: User Management

Click the Admin button in the top bar to open the Admin panel. The Users tab lets you manage who can access the app.

User Roles

RolePermissions
UserCan create tasks, save/load their own sessions, export XLSX
AdminAll user permissions + manage users and trades
Sys AdminAll admin permissions + view all sessions, configure Autodesk credentials

Adding a User

Click + Add User in the Users tab.
Enter a username, full name, email, password, and role.
Click Save User.

Editing a User

Click the Edit button on any row. Leave the password field blank to keep the existing password.

Deactivating a User

Edit the user and set Active to No. Deactivated users cannot log in but their session data is preserved.
